# EnlightenLM 实现状态总览

> **版本**：v3.0（模型无关外围监控架构）
> **重要提示**: 
> 1. **架构定位**：EnlightenLM 是一个模型无关的三层推理安全框架，安全能力完全由外围 L2/L3 提供
> 2. **设计原则**：不修改底层模型权重，通过 API 外围构建监控层
> 3. **当前实现**：v3.0 架构已完整实现，支持 Ollama、DeepSeek API 等多种后端

## 🎯 实际运行的代码路径

### 当前运行的核心文件（v3.0）
```
enlighten/
├── hybrid_architecture.py  (⭐ 实际使用的实现，支持本地模型和API模式)
├── api_server.py           (⭐ API服务，包含REST和WebSocket)
├── api/
│   ├── deepseek_client.py  (⭐ DeepSeek API客户端)
│   └── ollama_client.py    (⭐ Ollama客户端)

```

### 骨架代码（设计完整但作为参考）
```
enlighten/
├── l1_generation.py        (骨架 - L1透明调用设计参考)
├── l2_working_memory.py    (骨架 - L2工作记忆设计参考)
├── l3_controller.py        (骨架 - L3元控制器设计参考)
├── attention/              (骨架 - 注意力机制参考)
├── memory/                 (骨架 - 记忆管理参考)
├── audit/                  (骨架 - 审计系统参考)
├── cutoff/                 (骨架 - 截断机制参考)
└── adapters/               (骨架 - 模型适配器参考)
```

### 已废弃的设计（v2.5旧架构）
以下设计已在v3.0中废弃，不再是当前路线图的一部分：
- 双流注意力 (DAN + VAN 融合)
- DMN 动态记忆网络噪声抑制
- LSTM 风格遗忘门
- 记忆矩阵压缩与稀疏注意力选择

这些方向在远期若切换到 HuggingFace 原生模型加载模式时可作为研究探索，但不在当前路线图内。

## 💡 API模式下的近似策略

### 注意力权重近似
由于DeepSeek API是黑盒，不返回模型内部注意力，我们通过以下方式近似：
1. **词频/长度评分**: 长词/内容词权重高
2. **固定32维压缩**: 确保注意力历史维度一致
3. **平均池化**: 处理不同长度的输入

### 熵值计算
API模式下使用文本统计特征而非真实模型注意力熵：
```python
combined_entropy = (diversity * 0.4 +
                    (1 - repetition_rate) * 0.3 +
                    char_entropy * 0.3)
```

### 截断判据
仍使用文档中的三条件，但数据来源不同：
1. **低熵**: 基于文本统计，非真实注意力
2. **低方差**: 基于注意力历史变化
3. **下降趋势**: 基于多轮熵值变化

## 🚀 开发计划（对齐 v3.0 架构）

### 短期（1~4 周）

**1. 架构定位收拢与文档更新**
- ✅ 更新 implementation_status.md 至 v3.0
- 🔲 更新 api_reference.md，补充新接口
- 🔲 归档旧设计文档 (architecture.v2.5.md)

**2. 审计链激活**
- 将哈希链与 HMAC 签名模块接入实际推理管线
- 每个生成步骤记录 `(step, token, previous_hash)` 元组
- 响应中附带完整性签名

**3. L1 适配器稳定化**
- 解决 torch/numpy 数组混用的兼容性问题
- 确保 L1 适配器作为独立预处理模块正常工作

### 中期（1~3 个月）

**4. 偏见检测与缓解（P0）**
- 集成多领域敏感词库（性别、种族、职业等）
- 基于 AC 自动机实现高效匹配
- 在 L3 控制层增加偏见拦截规则

**5. 偏见检测与缓解（P1）**
- 构建有偏/无偏语句对比数据集
- 训练中间层线性偏见探针（需切换到支持输出 hidden states 的模型后端）
- 实现激活引导模块

**6. 干预策略精细化**
- 将现有的 halt 升级为三级策略：continue / halt / rewind
- 引入生成过程锚点（Engram Point）与回滚机制

### 长期（3~6 个月）

**7. Agent 协同验证**
- 集成多 Agent 辩论框架，作为 L3 后的二审"法官"

**8. 幻觉判别器 MLP**
- 收集标注数据（最少 500 组正常/幻觉样本）
- 离线训练轻量级 MLP 幻觉判别器

**9. 偏见与幻觉统一评估管线**
- 集成标准化基准（TruthfulQA、HaluEval、BBQ）
- 持续测量核心指标
