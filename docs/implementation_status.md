## 💡 API模式下的近似策略

### 注意力权重近似
由于DeepSeek API是黑盒，不返回模型内部注意力，我们通过以下方式近似：
1. **词频/长度评分**: 长词/内容词权重高
2. **固定32维压缩**: 确保注意力历史维度一致
3. **平均池化**: 处理不同长度的输入

### 熵值计算
API模式下使用文本统计特征而非真实模型注意力熵：
```python
combined_entropy = (diversity * 0.4 +
                    (1 - repetition_rate) * 0.3 +
                    char_entropy * 0.3)
```

### 截断判据
仍使用文档中的三条件，但数据来源不同：
1. **低熵**: 基于文本统计，非真实注意力
2. **低方差**: 基于注意力历史变化
3. **下降趋势**: 基于多轮熵值变化
