# EnlightenLM 实现状态总览

> **版本**：v3.0（模型无关外围监控架构）
> **重要提示**: 
> 1. **架构定位**：EnlightenLM 是一个模型无关的三层推理安全框架，安全能力完全由外围 L2/L3 提供
> 2. **设计原则**：不修改底层模型权重，通过 API 外围构建监控层
> 3. **当前实现**：v3.0 架构已完整实现，支持 Ollama、DeepSeek API 等多种后端

## 🎯 实际运行的代码路径

```
enlighten/

├── api/
```

### 已废弃的设计（v2.5旧架构）
以下设计已在v3.0中废弃，不再是当前路线图的一部分：
- 双流注意力 (DAN + VAN 融合)
- DMN 动态记忆网络噪声抑制
- LSTM 风格遗忘门
- 记忆矩阵压缩与稀疏注意力选择

这些方向在远期若切换到 HuggingFace 原生模型加载模式时可作为研究探索，但不在当前路线图内。

## 💡 API模式下的近似策略

### 注意力权重近似
由于DeepSeek API是黑盒，不返回模型内部注意力，我们通过以下方式近似：
1. **词频/长度评分**: 长词/内容词权重高
2. **固定32维压缩**: 确保注意力历史维度一致
3. **平均池化**: 处理不同长度的输入

### 熵值计算
API模式下使用文本统计特征而非真实模型注意力熵：
```python
combined_entropy = (diversity * 0.4 +
                    (1 - repetition_rate) * 0.3 +
                    char_entropy * 0.3)
```

### 截断判据
仍使用文档中的三条件，但数据来源不同：
1. **低熵**: 基于文本统计，非真实注意力
2. **低方差**: 基于注意力历史变化
3. **下降趋势**: 基于多轮熵值变化

## 🚀 开发计划（对齐 v3.0 架构）

### 短期（1~4 周）

**1. ✅ 信号自适应预处理模块（已完成）**
- 实现状态分类器（收敛/发散/离散）
- 实现 FFT 通道（收敛状态）
- 实现 Laplace 通道（发散状态）
- 实现 Z 通道（离散状态）
- 集成到 HybridEnlightenLM
- 添加 API 端点 `/l3/structured-features`

**2. 轻量级幻觉判别器训练**
- 收集标注数据（最少 500 组正常/幻觉样本）
- 训练轻量级 MLP 幻觉判别器
- 集成到 L3 控制器中

**3. ✅ 元认知自检提示（已完成）**
- 实现自动追问模型"你是否有偏见？"的机制
- 开发自我修正流程

**4. ✅ 监控面板优化（已完成）**
- 完善 Grafana 仪表板
- 添加实时告警机制

### 中期（1~3 个月）

**5. Agent 协同验证**
- 集成多 Agent 辩论框架（如 Markovian Multi-Agent Debate）
- 实现"实时监控（哨兵）+ 事后辩论（法官）"的双保险

**6. ✅ 干预策略精细化（已完成）**
- 将现有的 halt 升级为三级策略：continue / halt（重新采样并提示）/ rewind（回滚至上一个安全锚点）
- 引入生成过程锚点（Engram Point）与回滚机制

**7. ✅ 生产化部署（已完成）**
- 提供监控模块独立部署为 sidecar 服务
- 支持动态开启/关闭监控

### 长期（3~6 个月）

**8. 偏见与幻觉统一评估管线**
- 集成标准化基准（TruthfulQA、HaluEval、BBQ、CHiSafetyBench）
- 持续测量幻觉发现率、偏见引导有效率等核心指标

**9. 用户反馈闭环**
- 建立用户反馈→偏见/幻觉标记→模型更新闭环
- 开发反馈收集与分析工具

**10. 多语言支持**
- 扩展敏感词库到多语言
- 支持多语言模型的安全监控
