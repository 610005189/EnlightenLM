# EnlightenLM 用户使用手册

## 1. 项目简介

EnlightenLM 是一个基于认知神经科学的大模型安全推理与元认知框架，采用 L1/L2/L3 三层架构实现实时自我监控与安全截断。

**核心特性：**
- 三层架构设计（L1 生成层、L2 工作记忆层、L3 元控制器）
- 实时安全监控与截断机制
- 支持 DeepSeek API 作为底层模型
- 现代化的 Web 界面
- 高性能推理能力
- 支持 Markdown 格式渲染
- 主题切换功能（深色/浅色）
- 对话历史持久化存储

## 2. 系统架构

### 2.1 核心组件

| 组件 | 职责 | 位置 |
|------|------|------|
| L1 生成层 | 负责文本生成 | `enlighten/l1_generation.py` |
| L2 工作记忆层 | 管理上下文和注意力 | `enlighten/l2_working_memory.py` |
| L3 元控制器 | 监控和安全截断 | `enlighten/l3_controller.py` |
| API 服务器 | 提供 RESTful API | `enlighten/api_server.py` |
| 聊天界面 | 用户交互界面 | `docs/chat.html` |

### 2.2 技术栈

- **后端**：Python 3.8+、FastAPI、DeepSeek API
- **前端**：HTML5、CSS3、JavaScript
- **部署**：本地 HTTP 服务器

## 3. 快速开始

### 3.1 环境准备

1. **安装依赖**
```bash
pip install -r requirements.txt
```

2. **设置环境变量**
```bash
# Windows PowerShell
$env:DEEPSEEK_API_KEY = "your-api-key"

# 可选配置
$env:DEEPSEEK_MODEL_NAME = "deepseek-chat"
```

### 3.2 启动服务

1. **启动 API 服务器**
```bash
python -m enlighten.api_server
```

2. **启动聊天界面**
```bash
cd docs
python -m http.server 8080
```

3. **访问地址**
- 聊天界面：http://localhost:8080/chat.html
- API 文档：http://localhost:8000/docs

## 4. 功能使用

### 4.1 聊天界面操作

1. **发送消息**
   - 在输入框中输入文本
   - 按 Enter 键发送（Shift+Enter 换行）
   - 点击发送按钮

2. **快捷问题**
   - 新对话页面提供 4 个快捷问题
   - 点击即可发送对应的问题

3. **历史对话**
   - 左侧边栏显示历史对话
   - 点击可切换到对应对话
   - 点击"新对话"开始新的对话
   - 点击删除按钮可删除单个历史对话

4. **删除对话**
   - 点击顶部清除按钮会弹出确认框
   - 点击历史对话旁的删除按钮会弹出确认框
   - 确认后删除，取消则保持不变

5. **Markdown 渲染**
   - AI 生成的回复支持 Markdown 格式
   - 支持标题（# ## ###）
   - 支持粗体（**text**）和斜体（*text*）
   - 支持代码块和行内代码
   - 支持列表、引用、链接等

### 4.2 系统设置

1. **打开设置菜单**
   - 点击左下角"设置"按钮
   - 显示下拉菜单，包含"系统设置"选项

2. **主题切换**
   - 点击"系统设置"打开设置弹窗
   - 可选择深色或浅色主题
   - 主题选择自动保存
   - 刷新页面后保持主题设置

### 4.3 API 接口

**推理接口**：`POST /inference`

**请求参数**：
```json
{
  "text": "输入文本",
  "max_length": 2048,
  "user_params": {}
}
```

**响应格式**：
```json
{
  "session_id": "会话ID",
  "output": "生成的文本",
  "meta_description": "响应描述",
  "user_params": {},
  "cutoff": false,
  "cutoff_reason": null,
  "attention_stats": {"latency": 4.4}
}
```

**健康检查**：`GET /health`

## 5. 配置管理

### 5.1 配置文件

系统使用 YAML 配置文件，位于 `configs/` 目录：
- `balanced.yaml`：平衡模式（默认）
- `full.yaml`：完整模式
- `lightweight.yaml`：轻量模式

### 5.2 环境变量

| 变量名 | 描述 | 默认值 |
|--------|------|--------|
| DEEPSEEK_API_KEY | DeepSeek API 密钥 | 必须设置 |
| DEEPSEEK_MODEL_NAME | 模型名称 | deepseek-chat |

## 6. 故障排除

### 6.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| API 服务器启动失败 | DeepSeek API Key 未设置 | 设置 DEEPSEEK_API_KEY 环境变量 |
| 聊天界面无法连接 | API 服务器未运行 | 确保 API 服务器正在运行 |
| 响应延迟高 | 网络或 API 处理时间长 | 耐心等待，或检查网络连接 |
| 模型返回错误 | API 调用失败 | 检查 API Key 有效性，或稍后重试 |
| 历史对话消失 | localStorage 未正常工作 | 检查浏览器是否允许 localStorage |
| 主题不保存 | localStorage 被清除 | 重新设置主题 |

### 6.2 日志查看

- API 服务器日志：终端输出
- 浏览器控制台：F12 打开开发者工具

## 7. 性能优化

- **模型选择**：根据需求选择合适的模型
- **请求长度**：控制输入文本长度
- **批量处理**：合理设置 max_length 参数
- **网络优化**：确保网络连接稳定

## 8. 安全注意事项

- **API Key 保护**：不要在代码中硬编码 API Key
- **输入验证**：避免发送敏感信息
- **内容审核**：系统会进行安全监控，但仍需用户自行注意

## 9. 高级功能

### 9.1 自定义配置

创建 `configs/custom.yaml` 文件，修改以下参数：
- `model_name`：模型名称
- `max_tokens`：最大生成长度
- `temperature`：生成温度

### 9.2 多模型支持

系统支持通过配置文件切换不同的 DeepSeek 模型。
