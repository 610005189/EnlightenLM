# ADR-001: 模型无关的外围监控架构

## 状态

已接受

## 背景

EnlightenLM 项目的核心目标是提升大语言模型的安全性和可靠性，通过实时监控、幻觉检测与偏见缓解来实现。在项目初期，存在两种技术路线的选择：

1. **内部修改路线**：修改模型内部注意力机制（如双流注意力），让模型具备内在的自我监控回路
2. **模型无关路线**：在模型外部构建监控层，通过 API 调用实现安全监控

## 决策

我们选择**模型无关的外围监控路线**，具体决策如下：

1. **明确废弃内部修改代码**：将 `l1_generation.py` 中的双流注意力、DMN抑制、遗忘门等内部改造代码移至 `docs/archive/`
2. **坚持模型无关原则**：L1 层仅作为透明 API 调用，安全能力完全由 L2/L3 外围提供
3. **统一架构文档**：更新所有文档，明确说明模型无关的设计理念和实现方式

## 理由

选择模型无关路线的主要理由：

1. **通用性**：支持任何提供标准 API 的大模型，无需为每个模型定制适配
2. **可插拔性**：监控模块可独立开启/关闭，不影响主推理管线
3. **易于部署**：基于现有模型 API 构建，无需修改模型权重或内部结构
4. **可维护性**：架构清晰，组件职责明确，便于后续扩展和维护
5. **风险可控**：不修改模型内部，避免引入新的安全风险或性能问题

## 影响

1. **正面影响**：
   - 提高系统的通用性和可扩展性
   - 降低部署和维护成本
   - 便于集成到现有系统中
   - 减少对特定模型的依赖

2. **挑战**：
   - 无法直接访问模型内部注意力机制，需要通过近似方法获取相关信息
   - API 模式下某些功能（如 logits 级监控）可能受到限制
   - 需要通过外部信号和特征提取来实现对模型行为的监控

## 缓解策略

针对上述挑战，我们采取以下缓解策略：

1. **注意力权重近似**：通过词频/长度评分、固定维度压缩和平均池化等方法近似注意力权重
2. **熵值计算**：使用文本统计特征（多样性、重复率、字符熵）计算综合熵值
3. **信号自适应预处理**：通过状态分类和数学变换（FFT/Laplace/Z）提取结构化特征，提升监控效果
4. **轻量级 MLP 判别器**：基于离线标注数据训练轻量级模型，提高幻觉检测准确率

## 结论

模型无关的外围监控架构是当前阶段的最佳选择，它平衡了通用性、可维护性和安全性需求。通过不断优化外部监控方法和信号处理技术，我们可以在不侵入模型的前提下，实现对模型行为的有效监控和干预。

未来，随着技术的发展和需求的变化，我们可以重新评估是否需要突破模型无关原则，探索更深入的内部修改方案。
