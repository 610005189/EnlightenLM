## 🔴 P0 - 必须完成（阻塞后续开发）

### 1. 幻觉判别器训练数据收集

**任务描述**：收集至少500组正常/幻觉样本数据，用于训练轻量级MLP幻觉判别器。

**具体要求**：
- 每组数据包含：输入提示词、模型输出、标注标签（正常/幻觉）
- 需覆盖多种幻觉类型：事实性幻觉、编造引用、数字错误等
- 建议来源：TruthfulQA、HaluEval、或人工标注

**交付物**：
- [ ] 500+组标注数据（CSV或JSON格式）
- [ ] 数据字段说明文档

**对应开发任务**：`enlighten/memory/` 幻觉判别器模块训练

**负责人**：项目维护者

---

### 2. 模型后端决策

**任务描述**：决定是否切换到支持 hidden states 的模型后端，以启用偏见探针激活引导功能。

**选项分析**：

| 选项 | 支持Hidden States | 优点 | 缺点 |
|------|------------------|------|------|
| 继续使用当前API模式 | ❌ 不支持 | 无需更改 | 无法实现偏见探针 |
| 切换到 vLLM | ✅ 支持 | 高性能、支持hidden states | 需本地部署GPU |
| 切换到 HuggingFace Transformers | ✅ 支持 | 完整控制 | 资源消耗大 |

**决策项**：
- [ ] 确认是否需要偏见探针功能（P1优先级）
- [ ] 如需要，选择后端方案（vLLM / Transformers）
- [ ] 如不需要，可跳过此任务

**对应开发任务**：偏见探针激活引导模块

**负责人**：项目维护者

---

## 🟡 P1 - 重要（影响功能完整性）

### 3. 评估基准数据获取

**任务描述**：获取标准化评估基准数据集，用于验证系统对幻觉和偏见检测的效果。

**所需数据集**：

| 数据集 | 用途 | 获取方式 |
|--------|------|----------|
| TruthfulQA | 幻觉检测评估 | OpenAI/外部托管 |
| HaluEval | 幻觉生成与检测 | 公开数据集 |
| BBQ | 偏见基准测试 | 公开数据集 |
| CHiSafetyBench | 安全测试 | 公开数据集 |

**交付物**：
- [ ] 下载/获取各数据集
- [ ] 数据格式转换为项目兼容格式
- [ ] 建立 `tests/benchmarks/` 数据目录

**对应开发任务**：偏见与幻觉统一评估管线

**负责人**：项目维护者

---

### 4. Agent框架选型

**任务描述**：选择用于多Agent协同验证的辩论框架，作为L3之后的"二审法官"。

**可选框架**：

| 框架 | 描述 | 适用场景 |
|------|------|----------|
| Markovian Multi-Agent Debate | 马尔可夫多智能体辩论 | 事实核查 |
| GKMAD | Group澜Chat Multi-Agent Debate | 开放域讨论 |
| Self-Consistency + Debate | 自洽性+辩论混合 | 复杂推理验证 |

**决策项**：
- [ ] 选择辩论框架方案
- [ ] 确认是否需要联网检索能力
- [ ] 确定多Agent角色分工设计

**对应开发任务**：Agent协同验证模块

**负责人**：项目维护者
