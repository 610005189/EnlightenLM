# EnlightenLM 项目实施计划

## 一、项目概况

**项目目标**：构建一个通用、可靠、可审计的 AI 安全中间件，通过外围监控实现对 AI 生成过程的实时监控和干预，提升 AI 系统的可靠性和安全性。

**技术路线**：坚持模型无关的外围监控路线，通过 L2/L3 监控层实现对 L1 生成层的实时监控和干预。

## 二、实施阶段与时间线

### 阶段一：扎根"模型无关"范式（1-2个月）

#### 任务1：技术路线明确化
- **任务描述**：清理内部修改代码，明确技术路线
- **子任务**：
  1. 识别并移除以双流注意力为代表的内部修改代码到 `docs/archive/`
  2. 更新 README 和架构文档，明确声明模型无关路线
  3. 添加架构决策记录（ADR），记录关键技术选择的理由
- **责任人**：架构师
- **截止日期**：2周

#### 任务2：完成 MLP 幻觉判别器
- **任务描述**：实现轻量级 MLP 模型，用于预测幻觉风险
- **子任务**：
  1. 构建包含人工标注的幻觉/非幻觉 token 级标签的数据集
  2. 设计 MLP 架构，输入为 L2 提取的统计特征
  3. 训练和评估 MLP 模型
  4. 将 MLP 判别器集成到 L3 决策系统
  5. 实现幻觉判别器配置机制，支持通过配置文件和环境变量控制启用/禁用
- **责任人**：机器学习工程师
- **截止日期**：4周

#### 任务3：完善审计与可解释性
- **任务描述**：增强审计功能，实现可解释的决策过程
- **子任务**：
  1. 增强 Merkle 树审计功能，确保生成过程的完整性验证
  2. 实现干预决策的可解释日志
  3. 为每个干预行为提供详细的决策依据和风险评估
- **责任人**：后端工程师
- **截止日期**：3周

### 阶段二：探测"直接控制"路径（2-3个月）

- **截止日期**：6周

#### 任务5：研究可微分的置信度软约束

- **子任务**：
  1. 设计合规性惩罚项计算方法

- **责任人**：研究工程师
- **截止日期**：6周

## 七、预期成果

### 阶段一完成后
- 一个可投入生产的、能主动预测幻觉风险并分级干预的通用 AI 安全中间件
- 明确的技术路线和架构文档
- 完善的审计和可解释性功能
- 稳定的配置系统和测试流程

### 阶段二完成后
- 探索出在不侵入模型的前提下引导注意力的有效方法
- 评估软约束技术对文本质量的影响
- 为长期技术路线决策提供依据

## 八、风险评估

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| MLP 判别器性能不足 | 无法有效预测幻觉风险 | 增加训练数据量，优化模型架构，调整特征提取方法 |
| 动态前缀引导效果不佳 | 无法有效引导模型注意力 | 研究不同前缀设计策略，评估不同模型对前缀的敏感性 |
| 配置系统复杂度高 | 增加使用和维护成本 | 简化配置结构，提供详细文档和示例 |
| 测试覆盖不足 | 系统稳定性和可靠性风险 | 建立全面的测试套件，实现自动化测试流程 |

## 九、结论

EnlightenLM 项目通过明确技术路线、完成关键功能实现、优化系统架构，有望成为一个强大的 AI 安全中间件，为 AI 系统的可靠性和安全性做出重要贡献。团队应专注于阶段一的实施，确保核心功能的稳定和可靠，为后续的技术探索奠定坚实基础。
