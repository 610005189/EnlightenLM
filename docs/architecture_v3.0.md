# EnlightenLM 架构设计文档 v3.0

> **版本历史**  
> - v3.0 (2026-04-26): 收敛为模型无关的外围监控架构，补充信号自适应预处理模块，修正"无训练接入"表述，增加系统边界声明与审计链已知限制
> - v2.5 (早期): 包含双流注意力、遗忘门等 L1 内部设计（已废止，见附录）

## 1. 设计目标

EnlightenLM 是一个**模型无关**的推理安全框架。它不修改底层大模型的权重，也不侵入模型内部计算图，而是通过在模型外围构建 L2 工作记忆层和 L3 元控制层，实现对生成过程的**实时安全监控、幻觉截断与偏见缓解**。

核心能力：
- 实时检测并干预幻觉（事实性错误）
- 实时检测并缓解偏见（价值性偏差）
- 提供可审计、可回溯的生成记录
- 支持多种模型后端（Ollama、DeepSeek API、vLLM 等），无需修改模型

## 3. 分层详细设计

### 3.1 L1 生成层 —— 模型无关的透明调用

**定位**  
L1 仅负责向底层模型发送提示词并接收回复，**不进行任何内部计算图修改**。安全能力完全由外围 L2/L3 提供。

**支持的后端**
- Ollama (本地部署，通过 HTTP API 获取文本和 logits)
- DeepSeek API (云端，支持流式输出和 logprobs)
- vLLM (高性能推理引擎，支持返回 token 概率)
- 可扩展接口：任何实现 `generate(prompt, **kwargs) -> (text, logits)` 的后端均可接入

**数据出口**
- 纯文本序列
- 每个生成 token 的 logits / logprobs（后端需支持）
- （可选）如果后端提供 hidden states，可用于偏见探针（见 L3），但非必需

**约束**
- 不得要求修改模型权重
- 不得要求访问模型内部注意力矩阵（架构设计不依赖此信息）

### 3.2 L2 工作记忆层 —— 信号提取与状态维护

**功能模块**

1. **会话历史管理**
   - 维护对话轮次，存储原始提示词和模型回复
   - 提供时间窗口内的上下文摘要，供 L3 进行自指循环检测
   - 管理 Engram 锚点（安全截断后的回滚点）

2. **文本统计特征提取**
   - n-gram 重复率、词汇多样性 (TTR)
   - 句子长度、特殊字符比例
   - 这些特征作为幻觉风险的初步指标，与 logits 特征互补

3. **Logits 级实时监控**
   - 在生成循环中逐 token 获取模型输出的 logits
   - 计算：
     - **后验熵** \( H_t = -\sum p \log p \) （不确定性的实时指标）
     - **置信度** \( \text{conf}_t = \max(p) \) （模型对当前选择的把握）
     - **置信度比值** \( \text{margin}_t = \frac{\max(p)}{\text{second\_max}(p)} \) （决策边界清晰度）
   - 将时间序列特征输入信号自适应预处理层

4. **信号自适应预处理层（v3.1 新增）**

   **设计动机**

   原始信号（熵、置信度、重复率等）在不同生成状态下具有完全不同的语义。同一熵值在模型稳定推理时是正常的，在模型进入发散状态时可能是幻觉前兆。直接送入贝叶斯推断器会导致推断不稳定。

   **核心思路**

   > **收敛就用傅里叶，发散就用拉普拉斯，离散就用 Z**

   先对原始信号进行状态分类，再根据状态选择合适的数学变换进行特征提取，将结构化特征（而非原始噪音）送入 L3。

   **状态分类器**

   | 状态 | 行为特征 | 检测依据 |
   |------|----------|----------|
   | **收敛** | 熵持续下降，置信度稳定，输出模式呈现周期性 | 熵方差低、ΔH趋近于0、干预事件少 |
   | **发散** | 熵突然飙升，置信度骤降，分布趋于均匀 | 熵方差高、ΔH频繁为正、置信度骤降 |
   | **离散** | 状态在多个模式间跳跃，触发 halt/rewind | 窗口内干预事件 ≥ 2 次 |

   分类采用分层规则实现（可替换为轻量级 MLP）：

   ```
   输入：最近 W 步的信号窗口
   输出：状态标签 ∈ {CONVERGING, DIVERGING, DISCRETE}

   1. 若窗口内 halt/rewind 事件 ≥ 2 次 → DISCRETE
   2. 若 H(t) 方差 > τ_var 且 最近 3 步 ΔH > 0 → DIVERGING
   3. 若 H(t) 单调下降 且 C(t) 稳定 > τ_conf → CONVERGING
   4. 其他 → 默认 CONVERGING (保守假设)
   ```

   **变换通道**

   **收敛 → 傅里叶变换 (FFT)**

   - 对滑动窗口内的熵序列做 FFT，转换到频域
   - 提取前 k 个主频率分量的幅值和相位
   - 若高频分量能量占比异常升高 → 标记为"疑似周期性重复幻觉"
   - 计算开销：O(W log W)，窗口 W=32~64，可忽略

   **发散 → 拉普拉斯分布拟合**

   - 对最近窗口内的熵/置信度序列拟合 Laplace(μ, b) 分布
   - μ 取序列中位数，b 为平均绝对偏差（MLE 闭式解）
   - 计算当前值在拟合分布中的分位数
   - 若 b 突然增大且当前值落入尾部 → 标记为"高不确定性发散"
   - 计算开销：O(W)，简单闭式解

   **离散 → Z 变换**

   - 将干预事件序列编码为离散序列 e ∈ {0(continue), 1(halt), 2(rewind)}
   - 做 Z 变换，求传递函数的极点
   - 若主导极点模长 |z| > 0.95 → 干预效果边际递减，建议切换策略
   - 计算开销：多项式求根，窗口长度短（< 20 步）

   **设计原则**

   - 三个通道仅激活其一，计算开销可控
   - 模块可插拔：未启用时 L3 直接接收原始信号，保持向后兼容
   - 后续可用 MLP 分类器替代规则分类器，无需改动下游

5. **幻觉判别器（离线训练）**
   - 轻量级 MLP（输入维度 ≤ 8，隐藏层 64→32，输出二分类）
   - 训练数据：标注的正常推理 vs 幻觉轨迹（每类 ≥ 500 组）
   - 推理时前向传播计算幻觉风险概率 \(p_{\text{hallucination}} \in [0,1]\)
   - 模型权重存储为独立文件，推理时加载，不依赖 GPU

**状态输出**
- 每个生成步骤的实时特征向量（含预处理后的结构化特征）
- 幻觉风险分数
- 触发信号：高熵警报、周期性幻觉警报、高不确定性发散警报等

### 3.3 L3 元控制层 —— 安全决策与干预

L3 是系统的安全大脑，接收 L2 的监控信号，综合判断后发出干预指令。

**功能模块**

#### A. VAN 安全监控引擎

| 检测类型 | 方法 | 干预 |
|----------|------|------|
| 敏感词/有害内容 | 正则 + AC 自动机多模式匹配 | 立即 halt 或 rewind |
| 自指循环 | 检测模型输出中是否包含自身前文的异常引用 | halt + 提示重置 |
| 词汇过度重复 | 基于滑动窗口的 n-gram 频率 | 降低温度并重新采样 |
| 回答长度异常 | 过长或过短 | 标记风险，不强制截断 |

**智能冷却机制**：同一类型触发后，在一段冷却窗口内对同类事件降级处理，避免无限截断循环。

#### B. 贝叶斯病因推断器

将安全事件归类为两种病因：
- **先验偏见 (Prior Bias)**：模型固有的错误倾向，与输入无关。干预策略：降低温度，更多依赖外部证据。
- **感官噪声 (Sensory Noise)**：输入中存在误导信息或对抗性提示。干预策略：忽略噪声部分，要求模型基于内部知识重新生成。

推断依据：接收 L2 信号自适应预处理层输出的结构化特征，进行贝叶斯模型选择，输出病因概率分布。输入特征的改进（从原始信号到自适应变换后的结构化特征）显著提升了推断稳定性。

#### C. 偏见检测与缓解

- **P0 - 偏见词库与正则拦截**：预置多领域敏感词库（性别、种族、职业等），基于正则匹配硬过滤。触发后追加元认知自检提示。
- **P1 - 激活探针 + 引导**：若后端提供 hidden states，使用线性探针检测偏见方向，通过激活引导向量进行推理时校正。
- **元认知自检提示**：在敏感场景自动追加“Could you be wrong? 请反思是否有偏见”提示，引导模型自我修正。

#### D. 动态截断与干预引擎

三级干预策略（基于风险等级融合矩阵，综合考虑幻觉风险与偏见风险）：

| 风险等级 | 条件 | 动作 |
|----------|------|------|
| 🟢 低风险 | \(p_{\text{hall}} < 0.3\) 且无偏见触发 | continue（正常生成） |
| 🟡 中风险 | \(0.3 \le p_{\text{hall}} < 0.7\) 或偏见低置信度触发 | halt：暂停生成，以更低温度重新采样当前 token，或引入检索锚定 |
| 🔴 高风险 | \(p_{\text{hall}} \ge 0.7\) 或偏见高置信度触发 | rewind：回滚至最近的 Engram 锚点，切换至 greedy decoding，并触发审计记录 |

**温度调节**：连续置信度不足时，自动阶梯式降低温度（0.7 → 0.5 → 0.3）。

#### E. 审计与密码学完整性

- **哈希链**：每一步生成记录 `(step, token, previous_hash)`，最后生成链式哈希摘要。
- **HMAC 签名**：对整个生成会话的最终内容进行 HMAC 签名，防止事后篡改。
- **审计 API**：提供 `/audit/logs` 接口随时拉取带签名和哈希链的生成记录。

**已知限制**：审计链保证生成记录的完整性，但不做出内容质量判断。若攻击者绕过偏见/幻觉检测，审计链会忠实地记录被绕过后的有害输出。审计链的角色是"记录者"而非"裁判"，需配合其他安全模块使用。


### 3.4 系统边界与适用性声明

EnlightenLM 有效作用的范围：

| 能有效应对的 | 不能单独应对的 |
|-------------|---------------|
| 模型有能力避免但在推理中产生的幻觉 | 模型因训练数据缺失而根本不具备的知识（需补充检索增强） |
| 显式的偏见表述和刻板印象 | 深层编码在模型参数中的隐性偏见（需配合训练数据改进） |
| 推理过程中的错误自举和循环引用 | 对抗性攻击中对安全模块的定向绕过（需补充红队测试与对抗训练） |
| 生成状态的实时监控与干预 | 生成完成后的事后审查（需 Agent 辩论二审补充） |


## 4. 数据流与处理管线

```
用户输入 → Prompt 预处理
      ↓
L1 生成层 (调用后端 API)
      ↓  token by token (支持流式)
L2 工作记忆
  · 存储上下文
  · 计算文本特征
  · 提取 logits → 熵、置信度、比值
  · ★ 信号自适应预处理
       ├── 状态分类 (收敛/发散/离散)
       ├── 选择变换通道 (FFT/Laplace/Z)
       └── 输出结构化特征
  · 幻觉判别器 (MLP) → p_hall
      ↓
L3 元控制层
  · VAN 安全检测 (敏感词/自指/重复)
  · 偏见检测 (词库 + 可选探针)
  · 贝叶斯推断 → 病因分类 (基于结构化特征)
  · 综合风险矩阵 → 干预决策
      ↓
   ┌─ continue → 输出 token，进入下一步
   ├─ halt → 重新采样或追加提示
   └─ rewind → 回滚至锚点，改变解码策略
      ↓
最终输出 + 审计签名
```

## 5. 部署架构

- **推理服务**：模型后端独立部署（Ollama / vLLM / 云端 API）
- **EnlightenLM 服务**：作为 sidecar 或中间件运行，提供 REST API（Flask）
  - `/api/v1/chat/completions`：带安全监控的聊天接口
  - `/api/v1/audit/logs`：审计日志查询
  - `/api/v1/safety/config`：动态阈值调整
- **外部存储**：对话历史、审计链可持久化到 SQLite / PostgreSQL
- **Docker 化**：提供 Dockerfile 和 docker-compose，一键部署整栈

## 6. 设计原则

1. **模型无关**：不绑定特定模型，不修改权重，通过 API 构建安全层。
2. **可插拔**：监控模块可独立启停，偏见/幻觉检测可分别配置。
3. **分级干预**：根据风险等级选择继续、暂停或回滚，避免过度影响正常生成。
4. **轻量开销**：核心监控主要为文本特征和轻量 MLP，增量延迟可控（目标 <20%）。
5. **持续可审计**：每一步决策都可追溯，支持密码学签名。
6. **接口预留**：关键扩展点预留抽象接口，支持后续替换实现而不改动架构。


## 7. 工程口诀速查

| 口诀 | 数学工具 | 检测目标 | 工程实现 |
|------|----------|----------|----------|
| 收敛用傅里叶 | FFT | 周期性重复幻觉 | `np.fft.fft()` |
| 发散用拉普拉斯 | Laplace MLE | 高不确定性幻觉 | `b = mean(\|x - median\|)` |
| 离散用 Z | Z 变换/极点分析 | 干预效果评估 | 多项式求根 |

## 8. 已废弃设计（存档）

以下设计属于早期版本（v2.5），设想在 L1 层进行模型内部改造，但经工程评估发现与“模型无关”定位冲突，且无法在 API 模式下实现。已整体废弃，存档于 `docs/archive/architecture.v2.5.md`。

- 双流注意力 (DAN + VAN 融合)
- LSTM 风格遗忘门
- DMN 动态记忆网络噪声抑制
- 记忆矩阵压缩与稀疏注意力选择
- 动态温度/稀疏度的内部调节

这些方向在远期若切换到 HuggingFace 原生模型加载模式（可访问内部激活）时，可作为研究探索，但不在当前路线图内。

## 9. 未来发展

当前架构已为未来扩展留有接口：

- **信号自适应预处理**：状态分类器后续可替换为轻量级 MLP，变换通道可扩展新的数学工具。
- **偏见探针与激活引导**：当后端支持 hidden states 时，可在 L2 中直接插入线性探针，实现更精准的实时偏见抑制。
- **多 Agent 辩论验证**：L3 后可接入多 Agent 二审系统，对高风险输出进行交叉验证。
- **在线学习**：收集生产环境的误报/漏报数据，持续更新幻觉判别器和状态分类器。
