## 🎯 一句话概述

**EnlightenLM** 将大模型推理过程解耦为 **生成层（L1）**、**工作记忆层（L2）**、**元控制层（L3）**，借鉴人脑 DAN/VAN/DMN 注意力网络，通过工作记忆驱动的稀疏注意力和实时元控制，实现高效、可审计、可截断的安全推理。

---

## 🧠 核心思想：为什么需要“觉悟”？

当前大模型面临三大结构性困境：

| 困境 | 传统应对 | EnlightenLM 的解法 |
|------|----------|-------------------|
| **注意力不可控** | 提示词软引导，模型可能忽略 | **双流注意力 + 元控制器**：目标驱动（DAN）与刺激驱动（VAN）并行，L3 实时调节温度与稀疏度 |
| **自指递归与幻觉** | 依赖训练数据分布，无机制约束 | **DMN 抑制 + 遗忘门**：主动衰减内部噪声与 KV 缓存，截断自反循环 |
| **安全审计缺失** | 日志可篡改，无法追溯决策过程 | **密码学审计链 + 离线复盘**：每一步注意力状态、调控动作均被签名存证，事后生成可读报告 |

EnlightenLM 不替代 Transformer，而是为其装上一套 **“前额叶 + 刹车 + 黑匣子”**。

## 🔬 核心创新：人脑三层注意力网络的工程映射

| 人脑网络 | 功能 | EnlightenLM 对应组件 | 安全价值 |
|----------|------|---------------------|----------|
| **DAN** | 目标驱动的主动聚焦 | 任务偏置流 \(B_{\text{DAN}}\)，由 L3 下发 | 根据任务类型强制引导注意力，防止漂移 |
| **VAN** | 刺激驱动的自动重定向 | 显著性检测单元 + 中断掩码 \(M_{\text{VAN}}\) | 检测敏感词/异常模式，触发硬阻断 |
| **DMN** | 自发、自我参照的思维流 | 内部噪声 \(\xi\) + 遗忘门 \(f_t\) | 抑制无意义自循环，防止“注意力惯性” |
| **前额叶** | 元认知调控 | L3 元控制器（输出 \(\tau, \theta, \alpha, s, cutoff\)） | 实时调节聚焦/发散程度，动态截断 |

**核心公式（简化版）**：
\[
\text{Attn}_{\text{sparse}} = \text{softmax}\left(\frac{Q \tilde{K}^\top}{\sqrt{d}}\right) \tilde{V}
\]
\[
\text{Attn}_{\text{fused}} = g \cdot \text{Attn}_{\text{DAN}} + (1-g) \cdot \text{Attn}_{\text{VAN}}
\]
\[
\text{Output} = \text{renormalize}\left(\text{softmax}(\text{Attn}_{\text{fused}}/\tau) \cdot \mathbf{1}_{>\theta}\right) V + \alpha \cdot \xi
\]

## 📐 数学合理性验证摘要

经逐模块验证，框架满足自洽性、可微性与数值稳定性：

- **工作记忆稀疏注意力**：复杂度 \(O(n \cdot m \cdot d)\)，当 \(m \ll n\) 时远低于 \(O(n^2 d)\)；近似误差有界（\(m=512\) 时 \(\varepsilon<0.1\)）。
- **渐进双流**：稳定性标志 \(s\) 控制 DAN/VAN 计算频率，稳定时仅单流，节省 40% FLOPs。
- **元控制动态调节**：温度 \(\tau\) 与稀疏截断 \(\theta\) 有明确几何意义，硬约束防止极端值。
- **DMN 与遗忘门**：扰动经 LayerNorm 归一化，不累积；遗忘门提供指数衰减的 KV 缓存。
- **截断判据**：低注意力熵（\(\mu_H<0.5\)）+ 低方差（\(\sigma_H<0.05\)）+ 持续时间，结合 VAN 事件，有效识别自反循环。
- **审计密码学**：哈希链 + HMAC 提供标准抗篡改与存在性证明。

详细证明见 [`docs/math_verification.md`](docs/math_verification.md)。

## ⚙️ 工程可落地性评估

### 整体结论：**可行，路径清晰**

| 组件 | 可落地性 | 主要挑战 | 推荐策略 |
|------|---------|----------|----------|
| **HuggingFace 工作记忆+稀疏注意力** | ✅ 高 | 动态索引实现 | 原型首选平台 |
| **vLLM 生产级集成** | ⚠️ 中等 | 需定制 attention backend | 系统编程专家介入 |
| **双流注意力 (DAN+VAN)** | ⚠️ 中等 | FLOPs 增加 | 渐进式执行（稳定时单流） |
| **L3 实时熵监控与截断** | ✅ 高 | 轻量计算，可复用 L2 统计 | 直接实现 |
| **审计日志 + TEE 隔离** | ✅ 高 | 已有云服务支持 | 可独立先行构建 |
| **离线复盘服务** | ✅ 高 | 异步调用小模型 | 不阻塞推理 |

### 性能预期（相对标准 Transformer 7B）

| 指标 | 标准 Transformer | EnlightenLM | 变化 |
|------|------------------|-------------|------|
| 每 token 延迟 | 40ms | 56ms | +40% |
| 显存 | 14GB | 16GB | +2GB |
| 长上下文注意力计算 | \(O(n^2)\) | \(O(n \cdot 512)\) | 大幅降低 |
| 可审计性 | 无 | 密码级 | 新增 |
| 安全截断 | 无 | 实时 | 新增 |

## 🚀 实施路线图

### Phase 1：概念验证（当前阶段）
- **平台**：HuggingFace Transformers + 小型模型（如 Qwen2.5-7B）
- **目标**：实现 L1 稀疏注意力、L2 工作记忆、L3 规则控制器、基础审计日志
- **产出**：可运行的 Python 原型，验证核心思想

### Phase 2：增强 L3 与离线复盘
- **内容**：L3 集成截断逻辑，实现离线复盘服务（读取日志生成报告）
- **优势**：提供完整的“推理+审计+解释”闭环

### Phase 3：vLLM 深度集成
- **内容**：将工作记忆和稀疏注意力移植到 vLLM 后端
- **需求**：系统编程 / CUDA 优化经验
- **目标**：提供生产可用的 EnlightenLM 推理引擎

### Phase 4：云端 TEE 部署
- **内容**：将审计日志迁移至 GPU 机密计算实例（如阿里云、Azure）
- **目标**：实现硬件级安全隔离，完成最终架构形态
